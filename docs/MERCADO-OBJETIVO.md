# Mercado objetivo de AutonomIA

Este documento define los sectores comerciales prioritarios para vender AutonomIA como SaaS modular para negocios locales.

El objetivo no es vender todos los módulos a todos los sectores, sino priorizar aquellos negocios con más dolor operativo, más necesidad de presencia digital, más dependencia de reseñas y mayor valor en reservas, citas o leads.

## 2. Sectores prioritarios Nivel A

Los sectores Nivel A son los primeros recomendados para venta y piloto:

- Restaurantes.
- Bares.
- Cafeterías.
- Peluquerías.
- Centros de estética.
- Clínicas dentales.
- Fisioterapia.
- Gimnasios.
- Inmobiliarias.
- Abogados.

Motivo:

- Tienen necesidad frecuente de contenido.
- Dependen mucho de reseñas y reputación.
- Necesitan captar clientes o reservas.
- Suelen tener poco tiempo para gestionar redes, mensajes y seguimiento.
- El valor de una mejora pequeña puede justificar una cuota mensual.

## 4. Recomendación de ataque comercial

Primera oleada:

- Hostelería.
- Belleza.
- Salud.
- Inmobiliarias.

Motivo:

- Son sectores con dolor diario.
- Usan redes y reseñas de forma visible.
- Tienen tareas repetitivas fáciles de explicar.
- Pueden entender rápido el valor de SocialIA, ReviewIA, ReservaIA y LeadIA.

Segunda oleada:

- Profesionales.
- Automoción.
- Formación.
- Comercio.

Motivo:

- Tienen valor potencial, pero pueden requerir mensajes comerciales más específicos.
- En algunos casos la venta es más consultiva.
- La urgencia puede ser menor que en hostelería, belleza o salud.

## 5. Recomendación práctica para el piloto

Para los primeros 2 o 3 clientes reales:

- Elegir negocios cercanos y accesibles.
- Priorizar quien pueda dar feedback rápido.
- Vender el plan Crecimiento fundador.
- Limitar la promesa a SocialIA básico, Centro IA, dashboard, tareas y configuración de empresa.
- Presentar ReviewIA, WhatsAppIA, LeadIA y ReservaIA como roadmap, no como automatización real completa.

## 6. Mensaje comercial base

AutonomIA ayuda a negocios locales a ahorrar tiempo, publicar mejor, organizar tareas y preparar automatizaciones con IA sin tener que contratar herramientas separadas ni gestionar todo manualmente.

## 7. Riesgos comerciales

- Prometer automatizaciones reales antes de conectarlas.
- Vender a sectores demasiado distintos en el piloto.
- No definir claramente qué incluye el MVP.
- Subestimar soporte inicial.
- No medir coste de IA y APIs externas por cliente.
- No separar bien demo, trial y cliente de pago.

## 8. Siguiente decisión recomendada

Elegir 3 sectores para el primer piloto:

1. Hostelería.
2. Belleza.
3. Salud o inmobiliaria.

Después, adaptar landing, demos y mensajes comerciales a esos sectores antes de ampliar mercado.
